# Git Workflow and History Management

## Overview
This repository was created for Homework #1 to demonstrate practical usage of Git workflow and history management. The assignment focuses on maintaining a clean commit history, recovering lost work, and organizing repository versions.

## Tasks Performed
- Initialized a Git repository
- Added `project.txt` and created multiple commits
- Cleaned commit history by rewriting/squashing commits
- Simulated a lost commit and recovered it using Git tools
- Created a branch `recovered-commit` from the recovered commit
- Defined a custom Git alias for clearer commit history visualization
- Tagged the cleaned version of the project as `v1.0`
- Pushed the repository to GitHub
